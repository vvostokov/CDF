# `docs/references/` — Reference Documentation

References are **information-oriented**. Each reference answers: "What is X exactly?"

## Structure

<!-- Project-specific. Populate with API references, configuration references, etc. -->

* `api.md` — API reference.
* `configuration.md` — Configuration reference.
* `cli.md` — CLI reference (if any).
* `data-model.md` — Data model reference.

## Conventions

* Exhaustive, not narrative.
* Use tables where possible.
* Cross-link to guides (which explain how to use it).
* Cross-link to ADRs (which explain why it is the way it is).
