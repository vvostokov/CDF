# `docs/guides/` — How-to Guides

Guides are **task-oriented**. Each guide answers: "How do I do X?"

## Structure

* `user/` — for users of the system (how to use the system).
* `development/` — for contributors (how to develop).
* `operations/` — for operators (how to run, monitor, recover).

## Conventions

* Use second-person ("you").
* Use numbered steps for procedures.
* Use checklists for verifications.
* End each guide with **Next steps** and **Related**.
