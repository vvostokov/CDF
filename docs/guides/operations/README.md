# Operations Guides

> Guides for operating the system in production (or staging / local stack).

## What Goes Here

* Deployment.
* Monitoring.
* Incident response.
* Backup and recovery.
* Capacity planning.
* Security operations.

## Conventions

* Runbook format: Alert → Diagnose → Mitigate → Resolve → Post-mortem.
* Every runbook has been **executed** at least once (or has a note explaining why it has not).

## Templates

* `deployment.md`
* `monitoring.md`
* `incident-response.md`
* `backup-and-recovery.md`
* `capacity-planning.md`
* `security-ops.md`
