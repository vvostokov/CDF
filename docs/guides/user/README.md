# User Guides

> Guides for users of the system.

## What Goes Here

* How to install the system.
* How to configure it.
* How to use its main features.
* How to interpret errors.
* How to get help.

## Conventions

* Plain language. Avoid jargon.
* Screenshots where helpful.
* Use real examples (with placeholders for sensitive values).
* Cross-link to reference docs.

## Templates

* `getting-started.md` — the first doc.
* `feature-x.md` — one per major feature.
* `troubleshooting.md` — common problems and fixes.
