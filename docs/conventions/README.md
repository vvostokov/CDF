# `docs/conventions/` — Project Conventions

Stable rules the project follows. New contributors should read these first.

## Files

* `coding-style.md` — naming, formatting, file layout.
* `testing.md` — what kind of tests, when, where.
* `commits.md` — commit messages, branching, PR titles.
* `tooling.md` — which tools, versions, and why.

## How to Change

* Conventions change via PR.
* Changes are reviewed by at least one maintainer.
* A change to a convention must also update any tooling that enforces it.

---

> *Conventions are how a team moves as one body.*
