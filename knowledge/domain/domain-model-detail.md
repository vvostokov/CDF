# Domain Model — Detail Document

> Reserved file. Used to expand a specific aspect of the domain model in depth when the canonical `domain-model.md` would become unwieldy.

## Usage

* Create one file per detail topic, named `NNNN-topic.md`.
* Each file is a **deep dive** — usually accompanied by a diagram in `/architecture/diagrams/`.
* Link back from the canonical `domain-model.md`.

## Suggested Topics

* Aggregate lifecycle details
* Event schemas and evolution rules
* Consistency model and trade-offs
* Cross-context saga patterns
