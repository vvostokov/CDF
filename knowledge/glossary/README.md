# `glossary/` — Glossary Entries (Detailed)

## Purpose

Detailed entries for each glossary term. The summary in `/.cognition/memory/permanent/glossary.md` is the **short** form. The files here are the **long** form: full definition, examples, counterexamples, cross-references.

## File Naming

* `G-NNN-term.md`
* Numbering matches the summary.

## Template

See `0000-template.md` in this directory.

## Lifecycle

* Created when a term is added.
* Updated as the term's meaning evolves.
* Marked `deprecated` (not deleted) when retired.

## AI Behavior

When writing or reviewing a glossary entry, the AI must:

1. Provide both a short and long definition.
2. Give at least one example and one counterexample.
3. Identify the bounded context.
4. Link to the source.

---

> *A term exists only when both humans and code agree on what it means.*
