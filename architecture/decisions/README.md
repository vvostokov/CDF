# Decision Diagrams

> Visual companions to ADRs in `knowledge/architecture-decisions/`.

## How to Use

When an ADR is non-trivial, add a diagram here that illustrates the decision:

* Before/after comparison.
* State machine.
* Sequence diagram of the new flow.
* Component diagram showing the new boundary.

## Naming

* `NNNN-short-name.svg` (matching the ADR number).

---

> *A decision is a change of shape. The diagram captures the shape.*
