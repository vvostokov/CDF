# Trade-offs

> The trade-offs we accept and the trade-offs we refuse.

Every meaningful engineering decision is a trade-off. The skill is in knowing **which** trade-offs you are making — and being deliberate about them.

This document makes our trade-offs explicit so future contributors don't accidentally reverse them without realizing it.

## How to Use This Document

* When proposing a change, check whether it reverses an accepted trade-off. If so, **propose changing this document first**.
* When pressured to accept a refused trade-off, link to this document.
* Periodically (e.g. per quarter) review this document. Some accepted trade-offs may no longer be needed; some refused ones may now be wise.
