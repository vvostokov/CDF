# Domain Knowledge

> Stable facts about the **domain** the system models. Not about the system itself — about the world it operates in.

Examples:

* For a payroll system: tax brackets, deduction rules, payment-cycle conventions.
* For a medical system: clinical workflows, terminology standards, regulatory requirements.

## How to Use

* Each entry has an ID, a fact, an evidence link, and an owner.
* Entries are **dated** and not deleted — they are **retired** when no longer true.
* When the real world changes, this file changes before the code does.
