# Architecture Notes

> Free-form architectural thinking: hypotheses, sketches, alternatives considered, dead ends, lessons learned about the architecture itself.

This is the architect's **scratchpad** — but a disciplined one. Notes are versioned, dated, and either promoted to ADRs / the domain model or explicitly retired.

## Conventions

* Each entry has a heading, a date, and a status (`active`, `accepted`, `rejected`, `parked`).
* Each entry ends with one of:
  * **Promoted to:** ADR-XXXX
  * **Promoted to:** domain_model.md section X
  * **Rejected because:** …
  * **Parked until:** …
